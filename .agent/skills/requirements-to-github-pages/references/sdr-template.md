# SDR template

Use this structure for each Software Decision Record.

```md
# SDR-0001: [short title]

## Status
Accepted

## Date
YYYY-MM-DD

## Context
What requirements, constraints, risks, or repository realities created this decision?

## Decision
State the chosen approach clearly.

## Options considered
- Option A
- Option B
- Option C

## Consequences
What becomes easier, harder, safer, or more limited because of this choice?

## Requirements touched
List the requirement IDs, headings, or bullet items that materially influenced the decision.

## Rejected options and rationale
Explain briefly why the other options were not selected.
```

## Minimum SDR set for this skill

Create at least:
- one SDR for stack choice
- one SDR for client-side persistence choice
- one SDR for routing, state, and data model choices

Add more when the requirements justify it.
