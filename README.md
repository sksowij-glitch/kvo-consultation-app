# Система заявок на консультацію

[![Open in Browser](https://img.shields.io/badge/LIVE_DEMO-Open_App-success?style=for-the-badge&logo=googlechrome&logoColor=white)](https://sksowij-glitch.github.io/Kurbatov_V.O/)

Вебзастосунок для створення, редагування, відправки та відстеження запитів на індивідуальні консультації з викладачами. Проєкт розроблений у межах курсової роботи з дисципліни "Розробка, аналіз вимог та тестування вебзастосунків".

## 🌟 Ключові особливості

- **Повний цикл CRUD**: Створення, перегляд, редагування та видалення заявок.
- **Розумна валідація**: Перевірка дат (не в минулому, не вихідні), часу (08:30-18:00) та довжини тексту.
- **Багатомовність (i18n)**: Повна підтримка української та англійської мов із динамічним перемиканням.
- **Персоналізація**: Збереження нікнейму користувача, який автоматично додається до заявок.
- **Живий пошук та фільтрація**: Миттєвий пошук за темою, автором або предметом.
- **Темна та Світла теми**: Підтримка системних налаштувань та ручне перемикання.
- **Кастомні діалоги**: Заміна стандартних `confirm()` на стилізовані модальні вікна.
- **Секретні анімації**: Система ASCII-анімацій (пончик, матриця, гра життя), що активуються при взаємодії з логотипом.

## 🚀 Технології

- **Frontend**: Vanilla JS (ES6+), HTML5, CSS3.
- **Іконки**: Lucide Icons.
- **Зберігання**: `localStorage` для персистентності даних без бекенду.
- **Деплой**: GitHub Pages з автоматизацією через GitHub Actions.

## 📂 Структура репозиторію

- `index.html` — основна структура застосунку.
- `app.js` — логіка застосунку, i18n, обробка даних.
- `styles.css` — стилізація з використанням CSS Variables (Glassmorphism design).
- `requirements/` — специфікація вимог (Requirements.md).
- `docs/sdr/` — записи про архітектурні рішення (Software Decision Records).
- `data/` — демонстраційні дані в форматі JSON.

## 🛠️ Локальний запуск

1. Склонуйте репозиторій:
   ```bash
   git clone https://github.com/sksowij-glitch/Kurbatov_V.O.git
   ```
2. Відкрийте `index.html` у браузері.
3. **Важливо**: Для роботи функції "Скинути демо-дані" (завантаження з JSON) рекомендується використовувати локальний сервер (через розширення Live Server у VS Code або команду `npx serve .`), щоб уникнути обмежень CORS.
